# Atomic CSS Component Generator

> **One rule**: combine first letters of CSS property + value = class name.
> `display: flex` ‚Üí `df`, `justify-content: center` ‚Üí `jcc`, `align-items: center` ‚Üí `aic`

For the complete class reference, see [reference.md](reference.md).
**Do NOT guess class names ‚Äî even if you think you know the answer.** ALWAYS use MCP `lookup_class`, `validate_classes`, or `css_to_classes` to verify before outputting. No exceptions.

## Absolute Rules

1. **HTML only** ‚Äî never create separate CSS files
2. **Classes only** ‚Äî never use inline `style` attributes
3. **Grid first** ‚Äî use `dg` + `gtc` for layouts, not legacy float/flex grids
4. **Semantic tags** ‚Äî use `header`, `nav`, `main`, `section`, `article`, `footer`
5. **Responsive** ‚Äî always add `sm-`, `md-` prefixes for mobile/tablet
6. **4px spacing rhythm** ‚Äî all spacing values in multiples of 4
7. **HEX 6-digit uppercase** ‚Äî `cFFFFFF`, `bg000000`

## MCP Server (MANDATORY)

**ALWAYS use** the `atomic-css` MCP server ‚Äî even for classes you think you know. Confidence is not a substitute for verification.
- `lookup_class` ‚Äî class ‚Üí CSS mapping (verify before using)
- `validate_classes` ‚Äî batch validity check + suggestions for invalid classes
- `css_to_classes` ‚Äî convert CSS declarations ‚Üí Atomic classes (use in Step 1)
- `search_by_css` ‚Äî find classes by CSS property name

```json
{ "mcpServers": { "atomic-css": { "type": "sse", "url": "https://mcp.atomiccss.dev/sse" } } }
```

> **Why mandatory?** Class names are generated by code with subtle edge cases (unit ambiguity, shorthand expansion, special syntax). Human-like guessing produces plausible but wrong classes. The MCP server is the single source of truth.

## Core Naming Rule

**First letters of CSS property + value = class name.**

```
display: flex           ‚Üí df        justify-content: center ‚Üí jcc
align-items: flex-start ‚Üí aifs      flex-direction: column  ‚Üí fdc
white-space: nowrap     ‚Üí wsn       text-overflow: ellipsis ‚Üí toe
object-fit: cover       ‚Üí ofc       border-collapse: collapse ‚Üí bcc
table-layout: fixed     ‚Üí tlf       user-select: none       ‚Üí usn
pointer-events: none    ‚Üí pen       background-size: cover  ‚Üí bsc
```

This rule covers ALL CSS properties. Always verify with MCP ‚Äî do not rely on inference alone.

### Ambiguous Prefixes (unit changes meaning)
- `fs0` = `flex-shrink: 0` (no unit ‚Üí flex-shrink) vs `fs16px` = `font-size: 16px` (with unit ‚Üí font-size)
- `fw700` = `font-weight: 700` (number only, 100-900 in steps of 100)
- `o50` = `opacity: 0.5` (number 0-100, divided by 100)
- `zi10` = `z-index: 10` (number only)
- `order1` = `order: 1` (number only)

## Units

| Unit | Notation | Example |
|------|----------|---------|
| px | `px` | `w100px`, `gap8px` |
| % | `p` | `w50p` ‚Üí `width: 50%` |
| rem | `rem` | `fs1-5rem` ‚Üí `1.5rem` |
| em | `em` | `p1em` |
| vh/vw | `vh`, `vw` | `h100vh`, `w100vw` |
| vmin/vmax | `vmin`, `vmax` | `w50vmin` |
| fr | `fr` | `gtc1fr-2fr` |

> **rem base**: `html { font-size: 10px }` ‚Üí `1rem = 10px`

### Unit Size Convention
| Size | Unit | Example |
|------|------|---------|
| Under 20px | `px` | `gap8px`, `p12px`, `br4px` |
| 20px+ | `rem` | `gap2rem`(20px), `p3-2rem`(32px) |

## Hyphens & Special Notation

| Use | Example | Result |
|-----|---------|--------|
| Decimal | `gap1-5rem` | `1.5rem` |
| Value separator | `gtc1fr-2fr-1fr` | `1fr 2fr 1fr` |
| Media prefix | `sm-dn` | `@media(max-width:768px)` |
| Pseudo-class | `hover-bg000000` | `:hover` |
| `i` suffix | `w100pxi` | `!important` |
| `neg-` prefix | `neg-mt10px` | `margin-top: -10px` |

## Unit-Based Properties (prefix + value + unit)

These prefixes accept any numeric value with a unit:

| Prefix | Property | | Prefix | Property |
|--------|----------|-|--------|----------|
| `w`/`h` | width/height | | `m`/`mt`/`mr`/`mb`/`ml` | margin |
| `minw`/`maxw` | min/max-width | | `p`/`pt`/`pr`/`pb`/`pl` | padding |
| `minh`/`maxh` | min/max-height | | `gap`/`rg`/`cg` | gap/row-gap/column-gap |
| `t`/`r`/`b`/`l` | top/right/bottom/left | | `br` | border-radius |
| `fs` (with unit) | font-size | | `lh` | line-height |
| `ls` | letter-spacing | | `ws` | word-spacing |
| `ti` | text-indent | | `bw`/`btw`/`brw`/`bbw`/`blw` | border-width |
| `bs` | border-spacing | | `ow`/`oo` | outline-width/offset |
| `gac`/`gar` | grid-auto-columns/rows | | `gcs`/`gce`/`grs`/`gre` | grid-column/row-start/end |
| `sm`/`smt`/`smr`/`smb`/`sml` | scroll-margin | | `sp`/`spt`/`spr`/`spb`/`spl` | scroll-padding |
| `per` | perspective | | `in` | inset |
| `cw`/`crw` | column-width/rule-width | | `tuo`/`tdt`/`tbs` | text-underline-offset/decoration-thickness/tab-size |

### Axis Shortcuts
`px20px` = padding-left + padding-right, `py10px` = padding-top + bottom
`mx2rem` = margin-left + right, `my1rem` = margin-top + bottom, `mxa` = margin-left+right: auto

### Shorthand Expansion
`p`, `m`, `gap`, `bw` expand to longhand. So `p20px pl10px` works ‚Äî padding-left gets overridden.
Multi-value: `p10px-20px` ‚Üí top/bottom:10px, left/right:20px (2/3/4 values follow CSS rules)

## Complex Patterns (non-inferrable)

These patterns have special syntax that can't be guessed from the naming rule alone.

### Grid Templates
```
gtc1fr-2fr-1fr          ‚Üí grid-template-columns: 1fr 2fr 1fr
gtcr3-1fr               ‚Üí repeat(3, 1fr)
gtcrfit-minmax28rem-1fr  ‚Üí repeat(auto-fit, minmax(28rem, 1fr))
gtcrfill-minmax250px-1fr ‚Üí repeat(auto-fill, minmax(250px, 1fr))
gtcminmax100px-1fr       ‚Üí minmax(100px, 1fr)
gtccalc100p-200px-1fr    ‚Üí calc(100% - 200px) 1fr
gtrauto-1fr-auto         ‚Üí grid-template-rows: auto 1fr auto
```

### Grid Placement
```
gc-1_3      ‚Üí grid-column: 1 / 3
gc-span-2   ‚Üí grid-column: span 2
gr-2_4      ‚Üí grid-row: 2 / 4
ga-header   ‚Üí grid-area: header
ga-1_1_2_3  ‚Üí grid-area: 1 / 1 / 2 / 3
gta-header-header_sidebar-main_footer-footer ‚Üí grid-template-areas
```

### Border Shorthand
```
b1pxsolidDDDDDD        ‚Üí border: 1px solid #DDDDDD
bt2pxdashed000000      ‚Üí border-top: 2px dashed #000000
b1pxsolid255-0-0-50    ‚Üí border: 1px solid rgba(255,0,0,0.5)
```
Styles: solid, dashed, dotted, double, groove, ridge, inset, outset

### Individual Border Radius
`btlr8px` (top-left), `btrr8px` (top-right), `bblr8px` (bottom-left), `bbrr8px` (bottom-right)

### Shadow
```
bs2px4px10px0pxFF0000       ‚Üí box-shadow: 2px 4px 10px 0px #FF0000
bs0px4px12px0px0-0-0-10     ‚Üí box-shadow: ... rgba(0,0,0,0.1)
bsi0px2px4px0px000000       ‚Üí box-shadow: inset ...
ts2px-2px-4px-DDDDDD        ‚Üí text-shadow: 2px 2px 4px #DDDDDD (hyphen-separated)
```

### Text Stroke
`ts2pxFF0000` ‚Üí `-webkit-text-stroke: 2px #FF0000`

### Outline Shorthand
`ol2pxsolidFF0000` ‚Üí `outline: 2px solid #FF0000` (same format as border)

### Transition
```
tall200msease              ‚Üí transition: all 200ms ease
topacity500ms              ‚Üí transition: opacity 500ms
tall300mscb25-10-25-100    ‚Üí transition: all 300ms cubic-bezier(0.25,0.1,0.25,1)
tde300ms                   ‚Üí transition-delay: 300ms
tdu500ms                   ‚Üí transition-duration: 500ms
tpopacity_transform        ‚Üí transition-property: opacity, transform
```

### Transform (including 3D)
```
ttx10px / tty20px / ttz30px    ‚Üí translateX/Y/Z
tr45deg / trx90deg / try180deg ‚Üí rotate / rotateX/Y/Z
ts1-5 / ts1-5_2                ‚Üí scale(1.5) / scale(1.5, 2)
tskx10deg / tsky20deg          ‚Üí skewX / skewY
neg-ttx50p                     ‚Üí translateX(-50%)
```

### Animation
```
anfadeIn-500ms-ease             ‚Üí animation: fadeIn 500ms ease
anfadeIn-1s-ease-infinite       ‚Üí animation: fadeIn 1s ease infinite
anfadeIn-500ms-ease-3-200ms-forwards-reverse
```
**Preset keyframes**: fadeIn, fadeOut, slideUp, slideDown, slideLeft, slideRight, zoomIn, zoomOut, spinCW, spinCCW, bounce, pulse, shake, flip, swing, rubberBand, blink

### Gradient
```
bglg-to-r-FF0000-0000FF           ‚Üí linear-gradient(to right, #FF0000, #0000FF)
bglg-45deg-FF0000-00FF00-0000FF   ‚Üí linear-gradient(45deg, ...)
bgrg-circle-FF0000-0000FF         ‚Üí radial-gradient(circle, ...)
bgcg-from-0deg-FF0000-0000FF      ‚Üí conic-gradient(from 0deg, ...)
```
Color stops with position: `FF0000_50p` = `#FF0000 50%`, `transparent` keyword supported.

### Filter / Backdrop Filter
```
filterb5px     ‚Üí blur(5px)         bfb10px    ‚Üí backdrop-filter: blur(10px)
filterbr120p   ‚Üí brightness(120%)  filterg100p ‚Üí grayscale(100%)
filterhr90deg  ‚Üí hue-rotate(90deg) filterds2px4px8pxFF0000 ‚Üí drop-shadow(...)
```
Prefix: `filter` for filter, `bf` for backdrop-filter. Sub-functions: b(blur), br(brightness), c(contrast), g(grayscale), hr(hue-rotate), i(invert), o(opacity), sa(saturate), s(sepia), ds(drop-shadow)

### calc
```
wcalc100p-200px       ‚Üí width: calc(100% - 200px)
wcalc-add50p-100px    ‚Üí calc(50% + 100px)
wcalc-mul2rem-3       ‚Üí calc(2rem * 3)
wcalc-div100p-3       ‚Üí calc(100% / 3)
hcalc100vh-6rem       ‚Üí height: calc(100vh - 6rem)
```
Works with any unit-based property prefix (w, h, m, p, t, l, maxw, etc.)

### Line Clamp
`lc3-1-5rem` ‚Üí 3 lines, line-height: 1.5rem (adds overflow:hidden, display:-webkit-box, etc.)

### Aspect Ratio
`ar16-9` ‚Üí 16/9, `ar4-3` ‚Üí 4/3, `ar1` ‚Üí 1, `ara16-9` ‚Üí auto 16/9

### CSS Variables
`prefix--variable-name` ‚Üí `property: var(--variable-name)`
```html
<div class="bg--primary c--text gap--spacing br--radius">
```
Works with ALL prefixes, media queries, pseudo-classes, !important, before/after.
Shorthands decompose: `gap--x` ‚Üí row-gap + column-gap, `p--x` ‚Üí 4 directions.

## Pseudo-Classes (32)

Prefix any class: `{pseudo}-{class}`

**Interaction**: `hover-`, `focus-`, `active-`, `focus-within-`, `focus-visible-`
**Input**: `disabled-`, `enabled-`, `checked-`, `indeterminate-`, `default-`

**Link**: `link-`, `visited-`, `any-link-`, `target-`

## Pseudo-Elements

`before-{class}` ‚Üí `::before`, `after-{class}` ‚Üí `::after`
Use with `pr` (position:relative) on parent. `contentEmpty` sets `content:''`.

## Relationship Selectors

Pattern: `{parentClass}-{pseudo}-{combinator}-{targetTag}-{atomicClass}`

| Keyword | CSS | Example |
|---------|-----|---------|
| `child-{tag}-` | `> tag` | `card-hover-child-img-ts1-1` ‚Üí `.card:hover > img { transform: scale(1.1) }` |
| `children-{tag}-` | ` tag` | `nav-hover-children-a-cFFFFFF` ‚Üí `.nav:hover a { color: #FFF }` |
| `next-{tag}-` | `+ tag` | `input-focus-next-span-dn` ‚Üí `.input:focus + span { display: none }` |
| `siblings-{tag}-` | `~ tag` | `checkbox-checked-siblings-div-db` ‚Üí `.checkbox:checked ~ div { display: block }` |

The first segment (`card`, `nav`, `input`, `checkbox`) is the **parent element's class name**.

## Large Page Strategy (When Figma or design has many elements)

> **NEVER process an entire page at once.** When there are too many elements, context overflow causes the AI to fall back to guessing ‚Äî producing regular CSS classes instead of Atomic CSS. This is a critical failure mode.

### Always Split Into Sections

1. **Identify page structure first** ‚Äî break the design into logical sections:
   - Header/Nav, Hero, Product Grid, Features, CTA, Footer, etc.
2. **Process each section independently** ‚Äî apply the 3-step workflow below to each section separately
3. **Complete one section before starting the next** ‚Äî never process multiple sections simultaneously
4. **Assemble at the end** ‚Äî combine all sections into a single page

### Context Overflow Detection ‚Äî Split is MANDATORY when:
- Figma data contains 20+ elements
- User requests a full page or landing page
- Design contains multiple independent UI blocks

### Absolute Prohibitions
- **No guessing HTML** ‚Äî never write "this class is probably correct" without MCP verification
- **No regular CSS classes** ‚Äî `class="header"`, `class="card-title"` are NOT Atomic CSS
- **No `<style>` tags or CSS files** ‚Äî never create separate CSS under any circumstance
- **No assumptions for missing data** ‚Äî ask the user when Figma data is incomplete

## Mandatory 3-Step Workflow

> **You MUST follow all 3 steps in order. Skipping any step is a violation.**
> **For large pages, repeat these 3 steps for EACH section.**

### Step 1: MCP Lookup (BEFORE writing any HTML)

Identify the CSS properties you need, then query MCP to get the correct classes:

1. List the CSS properties needed for the component (layout, spacing, colors, typography, etc.)
2. Use `css_to_classes` to convert CSS declarations ‚Üí Atomic classes
3. For specific classes you're unsure about, use `lookup_class` to verify
4. For CSS properties you don't know the prefix for, use `search_by_css`

**Do NOT write HTML until you have MCP-verified classes for the component.**

### Step 2: Generate HTML

Using ONLY the MCP-verified classes from Step 1:

1. **Atomic CSS classes only** ‚Äî no inline styles, no CSS files
2. **Grid layout first** ‚Äî `dg` + `gtc` for page structure
3. **Responsive by default** ‚Äî add `md-` and `sm-` variants for key breakpoints
4. **Semantic HTML** ‚Äî proper tags (header, nav, main, section, article, footer)
5. **Hover/focus states** ‚Äî always add interaction feedback
6. **Transitions** ‚Äî `tall200msease` or `tall300ms` for smooth UX
7. **4px spacing rhythm** ‚Äî 4, 8, 12, 16, 20(2rem), 24(2-4rem), 32(3-2rem), 40(4rem)
8. **Under 20px ‚Üí px, 20px+ ‚Üí rem**

### Step 3: MCP Validation (BEFORE presenting to user)

1. **Collect ALL Atomic classes** from the generated HTML
2. **Run `validate_classes`** with the complete list ‚Äî fix any invalid classes using MCP suggestions
3. **Self-check** the following (no MCP needed):
   - Zero `style=` attributes
   - No `<style>` tags or external CSS
   - `sm-` or `md-` variants on key layout properties
   - Unit convention (under 20px ‚Üí px, 20px+ ‚Üí rem)
   - Spacing values are multiples of 4
   - HEX colors are 6-digit uppercase
   - Buttons/links have `hover-` and transition classes
   - Semantic HTML tags used

If MCP is unavailable, verify against [reference.md](reference.md) manually ‚Äî but this is a fallback, not the default.
