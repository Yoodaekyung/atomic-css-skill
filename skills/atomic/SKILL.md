# Atomic CSS Component Generator

> **One rule**: combine first letters of CSS property + value = class name.
> `display: flex` Ôćĺ `df`, `justify-content: center` Ôćĺ `jcc`, `align-items: center` Ôćĺ `aic`

For the complete class reference, see [reference.md](reference.md).
**Do NOT guess class names ÔÇö even if you think you know the answer.** ALWAYS use MCP `lookup_class`, `validate_classes`, or `css_to_classes` to verify before outputting. No exceptions.

## Absolute Rules

1. **HTML only** ÔÇö never create separate CSS files
2. **Classes only** ÔÇö never use inline `style` attributes
3. **Grid first** ÔÇö use `dg` + `gtc` for layouts, not legacy float/flex grids
4. **Semantic tags** ÔÇö use `header`, `nav`, `main`, `section`, `article`, `footer`
5. **Responsive** ÔÇö always add `sm-`, `md-` prefixes for mobile/tablet
6. **4px spacing rhythm** ÔÇö all spacing values in multiples of 4
7. **HEX 6-digit uppercase** ÔÇö `cFFFFFF`, `bg000000`

## MCP Server (MANDATORY)

**ALWAYS use** the `atomic-css` MCP server ÔÇö even for classes you think you know. Confidence is not a substitute for verification.
- `lookup_class` ÔÇö class Ôćĺ CSS mapping (verify before using)
- `validate_classes` ÔÇö batch validity check + suggestions for invalid classes
- `css_to_classes` ÔÇö convert CSS declarations Ôćĺ Atomic classes (use in Step 1)
- `search_by_css` ÔÇö find classes by CSS property name

```json
{ "mcpServers": { "atomic-css": { "type": "sse", "url": "https://mcp.atomiccss.dev/sse" } } }
```

> **Why mandatory?** Class names are generated by code with subtle edge cases (unit ambiguity, shorthand expansion, special syntax). Human-like guessing produces plausible but wrong classes. The MCP server is the single source of truth.

## Core Naming Rule

**First letters of CSS property + value = class name.**

```
display: flex           Ôćĺ df        justify-content: center Ôćĺ jcc
align-items: flex-start Ôćĺ aifs      flex-direction: column  Ôćĺ fdc
white-space: nowrap     Ôćĺ wsn       text-overflow: ellipsis Ôćĺ toe
object-fit: cover       Ôćĺ ofc       border-collapse: collapse Ôćĺ bcc
table-layout: fixed     Ôćĺ tlf       user-select: none       Ôćĺ usn
pointer-events: none    Ôćĺ pen       background-size: cover  Ôćĺ bsc
```

This rule covers ALL CSS properties. Always verify with MCP ÔÇö do not rely on inference alone.

### Ambiguous Prefixes (unit changes meaning)
- `fs0` = `flex-shrink: 0` (no unit Ôćĺ flex-shrink) vs `fs16px` = `font-size: 16px` (with unit Ôćĺ font-size)
- `fw700` = `font-weight: 700` (number only, 100-900 in steps of 100)
- `o50` = `opacity: 0.5` (number 0-100, divided by 100)
- `zi10` = `z-index: 10` (number only)
- `order1` = `order: 1` (number only)

## Units

| Unit | Notation | Example |
|------|----------|---------|
| px | `px` | `w100px`, `gap8px` |
| % | `p` | `w50p` Ôćĺ `width: 50%` |
| rem | `rem` | `fs1-5rem` Ôćĺ `1.5rem` |
| em | `em` | `p1em` |
| vh/vw | `vh`, `vw` | `h100vh`, `w100vw` |
| vmin/vmax | `vmin`, `vmax` | `w50vmin` |
| fr | `fr` | `gtc1fr-2fr` |

> **rem base**: `html { font-size: 10px }` Ôćĺ `1rem = 10px`

### Unit Size Convention
| Size | Unit | Example |
|------|------|---------|
| Under 20px | `px` | `gap8px`, `p12px`, `br4px` |
| 20px+ | `rem` | `gap2rem`(20px), `p3-2rem`(32px) |

## Hyphens & Special Notation

| Use | Example | Result |

## Unit-Based Properties (prefix + value + unit)

These prefixes accept any numeric value with a unit:

| Prefix | Property | | Prefix | Property |
|--------|----------|-|--------|----------|
| `w`/`h` | width/height | | `m`/`mt`/`mr`/`mb`/`ml` | margin |
| `minw`/`maxw` | min/max-width | | `p`/`pt`/`pr`/`pb`/`pl` | padding |
| `minh`/`maxh` | min/max-height | | `gap`/`rg`/`cg` | gap/row-gap/column-gap |
| `t`/`r`/`b`/`l` | top/right/bottom/left | | `br` | border-radius |
| `fs` (with unit) | font-size | | `lh` | line-height |
| `ls` | letter-spacing | | `ws` | word-spacing |
| `ti` | text-indent | | `bw`/`btw`/`brw`/`bbw`/`blw` | border-width |
| `bs` | border-spacing | | `ow`/`oo` | outline-width/offset |
| `gac`/`gar` | grid-auto-columns/rows | | `gcs`/`gce`/`grs`/`gre` | grid-column/row-start/end |
| `sm`/`smt`/`smr`/`smb`/`sml` | scroll-margin | | `sp`/`spt`/`spr`/`spb`/`spl` | scroll-padding |
| `per` | perspective | | `in` | inset |
| `cw`/`crw` | column-width/rule-width | | `tuo`/`tdt`/`tbs` | text-underline-offset/decoration-thickness/tab-size |

### Axis Shortcuts
`px20px` = padding-left + padding-right, `py10px` = padding-top + bottom
`mx2rem` = margin-left + right, `my1rem` = margin-top + bottom, `mxa` = margin-left+right: auto

### Shorthand Expansion
`p`, `m`, `gap`, `bw` expand to longhand. So `p20px pl10px` works ÔÇö padding-left gets overridden.
Multi-value: `p10px-20px` Ôćĺ top/bottom:10px, left/right:20px (2/3/4 values follow CSS rules)

These patterns have special syntax that can't be guessed from the naming rule alone.

### Grid Templates
```

gtcrfit-minmax28rem-1fr  Ôćĺ repeat(auto-fit, minmax(28rem, 1fr))
gtcrfill-minmax250px-1fr Ôćĺ repeat(auto-fill, minmax(250px, 1fr))
gtcminmax100px-1fr       Ôćĺ minmax(100px, 1fr)
gtccalc100p-200px-1fr    Ôćĺ calc(100% - 200px) 1fr
gtrauto-1fr-auto         Ôćĺ grid-template-rows: auto 1fr auto
```

### Grid Placement
```
gc-1_3      Ôćĺ grid-column: 1 / 3
gc-span-2   Ôćĺ grid-column: span 2
gr-2_4      Ôćĺ grid-row: 2 / 4
ga-header   Ôćĺ grid-area: header
ga-1_1_2_3  Ôćĺ grid-area: 1 / 1 / 2 / 3
gta-header-header_sidebar-main_footer-footer Ôćĺ grid-template-areas
```

### Border Shorthand
```
b1pxsolidDDDDDD        Ôćĺ border: 1px solid #DDDDDD
bt2pxdashed000000      Ôćĺ border-top: 2px dashed #000000
b1pxsolid255-0-0-50    Ôćĺ border: 1px solid rgba(255,0,0,0.5)
```
Styles: solid, dashed, dotted, double, groove, ridge, inset, outset

### Individual Border Radius
`btlr8px` (top-left), `btrr8px` (top-right), `bblr8px` (bottom-left), `bbrr8px` (bottom-right)

### Shadow
```
bs2px4px10px0pxFF0000       Ôćĺ box-shadow: 2px 4px 10px 0px #FF0000
bs0px4px12px0px0-0-0-10     Ôćĺ box-shadow: ... rgba(0,0,0,0.1)
bsi0px2px4px0px000000       Ôćĺ box-shadow: inset ...
ts2px-2px-4px-DDDDDD        Ôćĺ text-shadow: 2px 2px 4px #DDDDDD (hyphen-separated)
```

### Text Stroke
`ts2pxFF0000` Ôćĺ `-webkit-text-stroke: 2px #FF0000`

### Outline Shorthand
`ol2pxsolidFF0000` Ôćĺ `outline: 2px solid #FF0000` (same format as border)

### Transition
```
tall200msease              Ôćĺ transition: all 200ms ease
topacity500ms              Ôćĺ transition: opacity 500ms
tall300mscb25-10-25-100    Ôćĺ transition: all 300ms cubic-bezier(0.25,0.1,0.25,1)
tde300ms                   Ôćĺ transition-delay: 300ms
tdu500ms                   Ôćĺ transition-duration: 500ms
tpopacity_transform        Ôćĺ transition-property: opacity, transform
```

### Transform (including 3D)
```
ttx10px / tty20px / ttz30px    Ôćĺ translateX/Y/Z
tr45deg / trx90deg / try180deg Ôćĺ rotate / rotateX/Y/Z
ts1-5 / ts1-5_2                Ôćĺ scale(1.5) / scale(1.5, 2)
tskx10deg / tsky20deg          Ôćĺ skewX / skewY
neg-ttx50p                     Ôćĺ translateX(-50%)
```

### Animation
```
anfadeIn-500ms-ease             Ôćĺ animation: fadeIn 500ms ease
anfadeIn-1s-ease-infinite       Ôćĺ animation: fadeIn 1s ease infinite
anfadeIn-500ms-ease-3-200ms-forwards-reverse
```
**Preset keyframes**: fadeIn, fadeOut, slideUp, slideDown, slideLeft, slideRight, zoomIn, zoomOut, spinCW, spinCCW, bounce, pulse, shake, flip, swing, rubberBand, blink

### Gradient
```
bglg-to-r-FF0000-0000FF           Ôćĺ linear-gradient(to right, #FF0000, #0000FF)
bglg-45deg-FF0000-00FF00-0000FF   Ôćĺ linear-gradient(45deg, ...)
bgrg-circle-FF0000-0000FF         Ôćĺ radial-gradient(circle, ...)
bgcg-from-0deg-FF0000-0000FF      Ôćĺ conic-gradient(from 0deg, ...)
```
Color stops with position: `FF0000_50p` = `#FF0000 50%`, `transparent` keyword supported.

### Filter / Backdrop Filter
```
filterb5px     Ôćĺ blur(5px)         bfb10px    Ôćĺ backdrop-filter: blur(10px)
filterbr120p   Ôćĺ brightness(120%)  filterg100p Ôćĺ grayscale(100%)
filterhr90deg  Ôćĺ hue-rotate(90deg) filterds2px4px8pxFF0000 Ôćĺ drop-shadow(...)
```
Prefix: `filter` for filter, `bf` for backdrop-filter. Sub-functions: b(blur), br(brightness), c(contrast), g(grayscale), hr(hue-rotate), i(invert), o(opacity), sa(saturate), s(sepia), ds(drop-shadow)

### calc
```
wcalc100p-200px       Ôćĺ width: calc(100% - 200px)
wcalc-add50p-100px    Ôćĺ calc(50% + 100px)
wcalc-mul2rem-3       Ôćĺ calc(2rem * 3)
wcalc-div100p-3       Ôćĺ calc(100% / 3)
hcalc100vh-6rem       Ôćĺ height: calc(100vh - 6rem)
```
Works with any unit-based property prefix (w, h, m, p, t, l, maxw, etc.)

### Line Clamp
`lc3-1-5rem` Ôćĺ 3 lines, line-height: 1.5rem (adds overflow:hidden, display:-webkit-box, etc.)

### Aspect Ratio
`ar16-9` Ôćĺ 16/9, `ar4-3` Ôćĺ 4/3, `ar1` Ôćĺ 1, `ara16-9` Ôćĺ auto 16/9

### CSS Variables
`prefix--variable-name` Ôćĺ `property: var(--variable-name)`
```html
<div class="bg--primary c--text gap--spacing br--radius">
```
Works with ALL prefixes, media queries, pseudo-classes, !important, before/after.
Shorthands decompose: `gap--x` Ôćĺ row-gap + column-gap, `p--x` Ôćĺ 4 directions.

## Pseudo-Classes (32)

Prefix any class: `{pseudo}-{class}`

**Interaction**: `hover-`, `focus-`, `active-`, `focus-within-`, `focus-visible-`
**Input**: `disabled-`, `enabled-`, `checked-`, `indeterminate-`, `default-`
**Validation**: `valid-`, `invalid-`, `required-`, `optional-`, `in-range-`, `out-of-range-`
**Link**: `link-`, `visited-`, `any-link-`, `target-`
**Other**: `placeholder-shown-`, `empty-`, `read-only-`, `read-write-`, `fullscreen-`, `autofill-`, `modal-`, `playing-`, `paused-`, `picture-in-picture-`, `user-invalid-`, `user-valid-`

## Pseudo-Elements

`before-{class}` Ôćĺ `::before`, `after-{class}` Ôćĺ `::after`
Use with `pr` (position:relative) on parent. `contentEmpty` sets `content:''`.

## Relationship Selectors

Pattern: `{parentClass}-{pseudo}-{combinator}-{targetTag}-{atomicClass}`

| Keyword | CSS | Example |
|---------|-----|---------|
| `child-{tag}-` | `> tag` | `card-hover-child-img-ts1-1` Ôćĺ `.card:hover > img { transform: scale(1.1) }` |
| `children-{tag}-` | ` tag` | `nav-hover-children-a-cFFFFFF` Ôćĺ `.nav:hover a { color: #FFF }` |
| `next-{tag}-` | `+ tag` | `input-focus-next-span-dn` Ôćĺ `.input:focus + span { display: none }` |
| `siblings-{tag}-` | `~ tag` | `checkbox-checked-siblings-div-db` Ôćĺ `.checkbox:checked ~ div { display: block }` |

The first segment (`card`, `nav`, `input`, `checkbox`) is the **parent element's class name**.

## Mandatory 3-Step Workflow

> **You MUST follow all 3 steps in order. Skipping any step is a violation.**

### Step 1: MCP Lookup (BEFORE writing any HTML)

Identify the CSS properties you need, then query MCP to get the correct classes:

1. List the CSS properties needed for the component (layout, spacing, colors, typography, etc.)
2. Use `css_to_classes` to convert CSS declarations Ôćĺ Atomic classes
3. For specific classes you're unsure about, use `lookup_class` to verify
4. For CSS properties you don't know the prefix for, use `search_by_css`

**Do NOT write HTML until you have MCP-verified classes for the component.**

### Step 2: Generate HTML

Using ONLY the MCP-verified classes from Step 1:

1. **Atomic CSS classes only** ÔÇö no inline styles, no CSS files
2. **Grid layout first** ÔÇö `dg` + `gtc` for page structure
3. **Responsive by default** ÔÇö add `md-` and `sm-` variants for key breakpoints
4. **Semantic HTML** ÔÇö proper tags (header, nav, main, section, article, footer)
5. **Hover/focus states** ÔÇö always add interaction feedback
6. **Transitions** ÔÇö `tall200msease` or `tall300ms` for smooth UX
7. **4px spacing rhythm** ÔÇö 4, 8, 12, 16, 20(2rem), 24(2-4rem), 32(3-2rem), 40(4rem)
8. **Under 20px Ôćĺ px, 20px+ Ôćĺ rem**

### Step 3: MCP Validation (BEFORE presenting to user)

1. **Collect ALL Atomic classes** from the generated HTML
2. **Run `validate_classes`** with the complete list ÔÇö fix any invalid classes using MCP suggestions
3. **Self-check** the following (no MCP needed):
   - Zero `style=` attributes
   - No `<style>` tags or external CSS
   - `sm-` or `md-` variants on key layout properties
   - Unit convention (under 20px Ôćĺ px, 20px+ Ôćĺ rem)
   - Spacing values are multiples of 4
   - HEX colors are 6-digit uppercase
   - Buttons/links have `hover-` and transition classes
   - Semantic HTML tags used

If MCP is unavailable, verify against [reference.md](reference.md) manually ÔÇö but this is a fallback, not the default.
